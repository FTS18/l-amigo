Privacy Policy

Last Updated: February 8, 2026

TL;DR: L'Amigo stores your data locally on your device. We don't collect, transmit, or sell your personal information. Your GitHub token stays on your device. We only communicate with LeetCode and GitHub APIs using credentials you provide.

1. Introduction

L'Amigo ("we," "our," or "the Extension") is a Chrome browser extension designed to help you track your friends' LeetCode progress and sync your solutions to GitHub. This Privacy Policy explains how we handle your data.

By using L'Amigo, you agree to the practices described in this policy.

2. Data We Store Locally

All data collected by L'Amigo is stored locally on your device using Chrome's storage API (chrome.storage.local). We do not transmit this data to any external servers.

2.1 Personal Settings

  • Your LeetCode Username: Used to fetch your profile and submission data
  • GitHub Personal Access Token: Used exclusively for syncing submissions to your GitHub repository. This token is stored encrypted in your browser's local storage and never leaves your device.
  • GitHub Repository Name: The target repository for syncing your solutions
  • UI Preferences: Theme (dark/light mode), tab selections, and display settings

2.2 Friends Data

  • Friend Usernames: LeetCode usernames you choose to track
  • Cached Profile Data: Problem counts, difficulty distributions, streaks, topic statistics, language statistics, and recent submissions for each friend
  • Timestamps: Last fetch times to optimize API calls and implement smart caching

2.3 Submission Data

  • Accepted Submissions: Problem titles, IDs, difficulty levels, timestamps, and submission codes
  • Sync Status: Which submissions have been pushed to GitHub
  • Submission Calendar: 365 days of daily submission counts from LeetCode

3. External API Communication

L'Amigo communicates directly with third-party APIs to function. We do not act as an intermediary or store this data on any servers.

3.1 LeetCode GraphQL API

  • Purpose: Fetch user profiles, submission history, problem details, and submission calendars
  • Data Sent: LeetCode usernames, GraphQL queries, cookies for authentication
  • What We Receive: Public profile data as provided by LeetCode's API
  • Your Control: Only usernames you explicitly add are queried

3.2 GitHub REST API

  • Purpose: Create/update repositories and files with your accepted submission codes
  • Data Sent: Your GitHub token (from local storage), repository name, file contents (problem solutions)
  • Authorization: Uses the Personal Access Token (PAT) you provide. The token is sent directly from your browser to GitHub's API—never through our servers
  • Your Control: You control token permissions and can revoke access anytime from GitHub settings

4. Permissions Explained

L'Amigo requests the following Chrome permissions:

  • storage: Save your settings, friends list, and cached data locally
  • unlimitedStorage: Store large submission datasets without browser limits
  • alarms: Schedule background syncs and automatic profile refreshes
  • notifications: Notify you when syncs complete or when friends hit milestones
  • cookies: Access LeetCode cookies to authenticate API requests on your behalf
  • Host Permissions (leetcode.com, api.github.com): Communicate with LeetCode and GitHub APIs

5. What We Don't Do

L'Amigo does NOT:

  • ❌ Collect personally identifiable information (PII) beyond what you explicitly provide
  • ❌ Transmit your data to remote servers, analytics platforms, or third parties
  • ❌ Use tracking pixels, cookies, or analytics scripts
  • ❌ Sell, rent, or share your data with advertisers or data brokers
  • ❌ Monitor your browsing activity outside of leetcode.com
  • ❌ Access your GitHub repositories beyond what's needed for syncing (scoped by your token permissions)

6. Data Retention

All data is retained locally until you:

  • Manually delete it using the "Clear Data" button in Settings
  • Uninstall the extension (Chrome automatically removes extension storage)
  • Remove individual friends (their cached data is deleted)

We do not have access to or control over your local data.

7. Security

We implement security best practices:

  • Token Encryption: GitHub tokens are stored in Chrome's secure storage with encryption
  • HTTPS Only: All API communication uses encrypted HTTPS connections
  • No External Requests: Extension code only contacts LeetCode and GitHub APIs—no third-party scripts
  • Content Security Policy: Prevents injection attacks and unauthorized script execution

However, no system is 100% secure. Protect your GitHub token by:

  • Using fine-grained tokens with minimal permissions (only repo scope needed)
  • Setting token expiration dates
  • Revoking tokens immediately if you suspect compromise

8. Third-Party Services

L'Amigo interacts with:

  • LeetCode: Subject to LeetCode's Terms of Service and Privacy Policy
  • GitHub: Subject to GitHub's Terms of Service and Privacy Statement

We are not responsible for the privacy practices of these third-party services.

9. Children's Privacy

L'Amigo is not directed to children under 13. We do not knowingly collect data from children. If you are a parent/guardian and believe your child has used this extension, please contact us.

10. Changes to This Policy

We may update this Privacy Policy to reflect:

  • New features or functionality
  • Changes in legal requirements
  • Improvements to our privacy practices

Significant changes will be noted in the extension's changelog. Continued use after updates constitutes acceptance of the revised policy.

11. Your Rights

Since all data is stored locally on your device, you have full control:

  • Access: View all stored data via Chrome DevTools → Application → Storage → Extension
  • Modify: Change settings, remove friends, or update tokens anytime
  • Delete: Use "Clear Data" button or uninstall the extension
  • Export: Use the built-in export feature to download your data as JSON

12. Open Source Transparency

L'Amigo is open source. You can:

  • Review our source code on GitHub
  • Verify our privacy claims by inspecting the code
  • Contribute improvements or report issues
  • Audit network requests using Chrome DevTools
